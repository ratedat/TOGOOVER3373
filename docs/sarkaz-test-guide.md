# RHODES OBS COMMANDER3373 サルカズADB取得テスト手順

この配布版は、統合戦略#5「サルカズの炉辺奇談」のADB/OCR取得を優先して確認するテスト版です。他の統合戦略の表示や手入力機能は残っていますが、今回のテスト対象はサルカズのみです。

## 必要なもの

- Windows x64 PC
- MuMu Player 12 などADB接続できるAndroidエミュレーター
- アークナイツ日本版
- MuMu側でサルカズの探索中マップ画面まで進めた状態

## 起動

1. `RHODES OBS COMMANDER3373-0.1.0-x64.exe` を実行します。
2. Windows SmartScreenが出た場合は、内容を確認してから `詳細情報` -> `実行` を選びます。
3. 初回起動時の保存先は、迷ったら `実行ファイル側` を選んでください。EXEの横に `RHODES OBS COMMANDER3373 Data` フォルダが作られ、削除しやすくなります。
4. ポート番号は通常そのままで構いません。既に使われている場合だけ別番号にしてください。

## 初期設定

1. 上部メニューまたは画面内ボタンから `OBS設定` を開きます。
2. ADB設定で `自動検出` を押します。
3. MuMuが見つからない場合は、ADBパスにMuMuの `adb.exe` を指定します。例: `M:\Program Files\Netease\MuMu Player 12\shell\adb.exe`
4. `接続テスト` を押し、接続先デバイスが表示されることを確認します。
5. `スクリーンショットテスト` を押し、現在のMuMu画面が取れることを確認します。

## サルカズで確認してほしいこと

1. `共通設定` で統合戦略を `IS#5 サルカズの炉辺奇談` にします。
2. MuMu側を探索中のマップ画面にします。
3. ADB取得から以下を試します。
   - `基本情報`: 等級、分隊、希望、希望上限、源石錐、耐久値、シールド、指揮Lv、構想、時代
   - `秘宝`: 所持秘宝。3行以下ではスクロールせず、4行目以降がある場合だけスクロールします。
   - `思案`: 所持思案
   - `オペレーター`: 招集済みオペレーター
4. 取得中はMuMuやアプリを触らず、完了まで待ってください。
5. 結果がおかしい場合は、アプリ下部のADB取得ログと保存ログのパスを確認してください。

## OBS表示

- OBS Browser Sourceには `Overlay` のURLを使います。
- 例: `http://127.0.0.1:<アプリに表示されたポート>/overlay`
- 今回はADB取得の精度確認が主目的です。OBSレイアウトは今後調整します。

## 不具合報告でほしい情報

- 何を取得しようとしたか: 基本情報 / 秘宝 / 思案 / オペレーター / 時代
- 実際のゲーム画面スクリーンショット
- アプリに表示された取得結果
- ADB取得ログの保存パス、またはログJSON
- 使用エミュレーターと解像度

## 既知の注意

- このテスト版はサルカズ優先です。他ISのADB取得精度は保証しません。
- 取得結果はOCRに依存するため、画面演出や重なりで誤認識する場合があります。
- ADB操作はタップ/スワイプ位置を矩形内でランダム化しています。
- Android Backキーは使わず、基本的にタップで開閉します。
